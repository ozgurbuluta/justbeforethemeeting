How this app uses Google user data

Just Before The Meeting is a macOS menu bar application. If you choose to sign in with Google, the app accesses your Google account data only as described below, in line with the Google API Services User Data Policy. The app does not use Google user data to train generalized artificial intelligence or machine learning models.

Data accessed

The app requests this OAuth scope:

  • https://www.googleapis.com/auth/calendar.readonly — read-only access to Google Calendar.

Through the Google Calendar API, the app may access:

  • Calendar list metadata — information returned by the Calendar API for calendars in your account (for example calendar identifiers and whether a calendar is selected or hidden), so the app knows which calendars to read.
  • Event metadata for events in a rolling time window (up to approximately the next 48 hours), including event identifiers, titles or summaries as returned by Google, start and end times, and links the API may provide (such as htmlLink and hangoutLink) where present on an event.

The app does not request additional Google scopes. It does not read your email, contacts, Drive files, or other Google product data outside read-only Calendar access as described above.

Data usage

Google user data accessed by the app is used only to provide the app’s features on your Mac, including:

  • Periodically refreshing calendar information according to the poll interval you configure.
  • Applying your in-app filters and rules to determine which meetings matter for alerts.
  • Showing countdowns, menu bar status, and optional local notifications before meetings.

We do not use this data for advertising, sale, or user profiling. We do not build advertising profiles from Google user data.

Data sharing

The app does not send your Google calendar content or OAuth tokens to servers operated by the app developer. There is no developer-controlled backend that collects Google user data from the app.

Data flows that involve third parties:

  • Google — Your device communicates directly with Google’s OAuth and Google Calendar API endpoints over HTTPS when you sign in and when the app refreshes calendar data.
  • GitHub Pages — This marketing and documentation website (including this privacy policy) is static content; it does not receive your calendar data or OAuth tokens from the app. GitHub’s processing of site visitors is described in GitHub’s privacy statement.

Storage and protection

OAuth tokens (access and refresh tokens) are stored in the macOS Keychain on your computer, using the system’s secure storage APIs. Calendar and event information used by the app is held in memory on your device while the app runs, to drive countdowns and the user interface.

All requests to Google’s APIs use HTTPS.

Retention and deletion

OAuth tokens remain on your Mac in the Keychain until you sign out in the app or remove the app’s data. Signing out in the app removes stored OAuth tokens from the Keychain on that Mac.

You can also revoke the app’s access to your Google account at any time:

  • Google Account → Third-party connections — remove access for Just Before The Meeting (or the OAuth client name shown in your account).

For questions or additional deletion-related requests regarding this app, you may open an issue on the project repository: github.com/ozgurbuluta/justbeforethemeeting.

The macOS application (summary)

Sign-in uses Google OAuth with PKCE and a local loopback redirect in the browser. When you use Google services, Google’s privacy policy applies to data processed by Google: https://policies.google.com/privacy

This website

This site is static (download and product information). It does not set analytics cookies as part of the repository’s default files. If the site is hosted on GitHub Pages, GitHub’s policies and processing may apply; see GitHub’s privacy statement.
